6. 우선순위 — 시작 전 반드시 박아둘 것

  1. 무료, 즉시 적용
    클라이언트 16kHz mono Opus 64kbps 인코딩
    업로드 1/10 + 서버 리샘플링 제거 → 서버 CPU 25%↓. 글로벌 latency 흡수 최대 효과. record 패키지 옵션 한 줄 추가. audio-regression 으로 회귀 검증 필수 (Task #42).
  2. 방어용
    업로드 캡 + 레이트리밋 + R2 lifecycle
    FastAPI middleware max_bytes=2MB, slowapi 분당 10회 IP, R2 60일 무접속 자동삭제. 비용 폭주 + 어뷰즈 동시 방어 (Task #43).
  3. 선택, 추후
    WAV 해시 → 노트 캐시
    정상 사용 hit rate 5–10% (재시도/봇 한정). 처리량 증가는 적지만 어뷰즈 방어막. DAU 증가 후 도입 검토.
  4. 신중 검토
    pYIN 알고리즘 튜닝
    frame_length / hop_length / 다운샘플링. 처리량 30–50% 증가 가능하지만 정확도 회귀 위험. audio-regression 베이스라인 확보 + 사용자 영향 평가 필요. MVP 외 별도 의사결정.

10. 서비스 개시 UX 모델 (CapCut 참고)

편집/로컬 저장 = 무료, 내보내기/클라우드 동기화 = 유료. 워터마크/광고 없음. 음악 결과물 품질 무차별.

핵심 동선

로그인이 트리거되는 4가지 순간

  1. 내보내기 버튼 탭 → paywall → 로그인 → IAP → export
  2. "다른 기기에서 불러오기" → paywall → 로그인 → 클라우드 동기화
  3. "이 디바이스 외 다른 곳에서도 사용" 또는 백업 의도 → paywall → 로그인 → R2 업로드
  4. 회원 탈퇴 (= 결제한 유저만 해당. 비로그인은 그냥 앱 삭제)
→ 별도의 "로그인" 단독 진입점 없음. 모든 로그인은 paywall flow 의 일부. 약관 동의도 paywall 시트 내 inline 문구로 갈음.
